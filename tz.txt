## Проект

**«Система планирования культурных маршрутов на основе базы данных объектов культурного наследия города Москвы»**

---

# **1. Назначение системы**

Система предназначена для автоматизированного формирования **пешеходных и туристических маршрутов по объектам культурного наследия** на основе открытых государственных данных города Москвы.

Система должна помогать:

* туристам,
* жителям города,
* исследователям городской среды

выбирать оптимальные маршруты для знакомства с историческим и культурным наследием столицы.

---

# **2. Цель разработки**

Создать клиентоориентированное веб-приложение, которое:

* получает от пользователя точку начала маршрута;
* автоматически выбирает культурные объекты поблизости;
* формирует логически связанный маршрут;
* визуализирует его на интерактивной карте;
* предоставляет справочную информацию по каждому объекту.

---

# **3. Исходные данные**

Используется официальный открытый государственный датасет:

**«Объекты культурного наследия города Москвы»**

Состав данных:

* наименование объекта;
* адрес;
* район;
* тип;
* год постройки;
* уровень охраны;
* описание;
* географические координаты.

---

# **4. Технические требования**

## 4.1. Архитектура

Система должна быть реализована в архитектуре:

> **Frontend – Backend – Database**

```
Браузер пользователя  
→ Web-интерфейс  
→ REST API  
→ Сервер приложения  
→ MySQL  
```

---

## 4.2. СУБД

Используется:

> **MySQL 8.x**

Обязательно применение:

* типа `POINT`
* spatial-индексов
* функций:

  * `ST_Distance_Sphere`
  * `ST_Within`
  * `ST_Buffer`

---

# **5. Структура базы данных**

Нейросеть должна спроектировать и реализовать БД, содержащую не менее следующих таблиц:

## 5.1. Таблица объектов наследия

Хранит данные открытого датасета.

Поля:

* id
* name
* address
* district
* type
* build_year
* description
* heritage_level
* location (POINT)

Должен быть создан:

```sql
SPATIAL INDEX(location)
```

---

## 5.2. Таблица пользователей

Хранит данные зарегистрированных пользователей.

---

## 5.3. Таблица маршрутов

Хранит:

* пользователя
* точку старта
* дату построения

---

## 5.4. Таблица связей маршрута и объектов

Хранит последовательность посещения объектов.

---

# **6. Обработка открытых данных**

Нейросеть должна:

1. Загрузить CSV или JSON с портала открытых данных
2. Очистить поля
3. Преобразовать координаты
4. Сохранить в MySQL
5. Поддерживать возможность повторного обновления

---

# **7. Алгоритм построения маршрута**

Алгоритм должен:

1. Получать точку старта пользователя
2. Выбирать N ближайших культурных объектов с помощью MySQL
3. Строить маршрут, минимизируя расстояние

Использовать:

* расстояния по сфере (ST_Distance_Sphere)
* графовую модель:

  * вершины — объекты
  * рёбра — расстояния между ними

Применить:

> Жадный алгоритм обхода или упрощённую задачу коммивояжёра

---

# **8. API**

Нейросеть должна реализовать REST-API:

| Метод | URL           | Назначение        |
| ----- | ------------- | ----------------- |
| GET   | /api/objects  | получить объекты  |
| POST  | /api/route    | построить маршрут |
| POST  | /api/register | регистрация       |
| POST  | /api/login    | авторизация       |
| GET   | /api/routes   | история маршрутов |

---

# **9. Пользовательский интерфейс**

Frontend должен включать:

* интерактивную карту
* ввод адреса или клик по карте
* кнопку «Построить маршрут»
* список объектов маршрута
* карточки объектов
* визуализацию маршрута линиями

Использовать:

* Leaflet
* SVG-иконки
* AJAX

---

Нейросеть обязана:
